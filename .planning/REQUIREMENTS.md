# Requirements — Дашборд «Вплив кешбеків»

## v1 Requirements

### Navigation
- [x] **NAV-01**: «Топ-менеджмент» у sidebar стає collapsible-групою з двома підпунктами: «Загальний ROI» (`/dashboard/exec`) та «Вплив кешбеків» (`/dashboard/exec/cashback-impact`)

### KPI Summary
- [ ] **KPI-01**: Картка «Конверсія активації кешбеку» — % нових клієнтів з обраними категоріями протягом 24 год після отримання картки
- [ ] **KPI-02**: Картка «Час до першої транзакції» — середній час від вибору категорії до першого нарахування кешбеку (у днях)
- [ ] **KPI-03**: Картка «Reactivation Rate» — % «сплячих» клієнтів (>90 днів без транзакцій), що повернулись після оновлення програми
- [ ] **KPI-04**: Картка «Транзакційність» — середня кількість транзакцій/місяць у групі з кешбеком vs без кешбеку
- [ ] **KPI-05**: Картка «Зміна середнього чека» — порівняння середнього чека в категоріях з кешбеком до/після підключення програми (у %)
- [ ] **KPI-06**: Картка «Credit Utilization» — частка транзакцій за рахунок кредитного ліміту в категоріях кешбеку (у %)

### ROI по категоріях (Блок 1)
- [ ] **ROI-01**: Горизонтальний bar chart усіх ~17 категорій, відсортований за ROI від найвищого до найнижчого, з кольоровим зонуванням фону: червона зона ROI <50x, жовта 50–200x, зелена >200x
- [ ] **ROI-02**: Scatter plot «Оборот vs Кешбек» — кожна категорія = бульбашка, розмір бульбашки = кількість транзакцій; аномальні категорії (великий оборот + малий кешбек) підсвічені tooltip-ом
- [ ] **ROI-03**: Line chart динаміки ROI топ-5 категорій за останні 6 місяців (мультилінійний)

### Конверсійна воронка (Блок 2)
- [ ] **CONV-01**: Funnel-діаграма з можливістю перемикання між категоріями: показує кроки «Всього транзакцій → Побачили офер → Обрали категорію → Активували кешбек» з % конверсії на кожному кроці
- [ ] **CONV-02**: Теплова карта «Конверсія vs Середній чек» — grid де рядки = категорії, колонки = діапазони чеків, колір комірки = % конверсії; виявляє де висока конверсія при низькому чеку (дешева лояльність)
- [ ] **CONV-03**: Виділений блок/таблиця з категоріями нульової або критично низької (<5%) конверсії з поясненням можливих причин (проблема оферу vs проблема комунікації)

### Інкрементальність (Блок 3)
- [ ] **INC-01**: Grouped bar / before-after chart по категоріях: оборот і кількість транзакцій за 3 місяці до vs 3 місяці після запуску оферу
- [ ] **INC-02**: Cohort line chart — 2 криві за 6 місяців: клієнти що активували кешбек vs контрольна група (без активації); метрика = середній оборот на клієнта
- [ ] **INC-03**: Stacked/grouped chart «канібалізація» — чи зросли витрати сумарно чи просто перерозподілились між категоріями при активації кешбеку

### UX / Фільтри
- [ ] **UX-01**: Select-фільтр по категорії у header сторінки (впливає на CONV-01, INC-01)
- [ ] **UX-02**: Select-фільтр по периоду «Останні 6 місяців / 3 місяці / З початку року» у header (впливає на всі часові чарти)
- [ ] **UX-03**: Всі дані — моки у файлі сторінки, реалістичні цифри по ~17 категоріях кешбеку

## v2 Requirements (Deferred)

- **PORT-01**: 2×2 матриця «Конверсія vs ROI» з 4 квадрантами та action items
- **PORT-02**: Топ-3 категорії для інвестування і топ-3 для перегляду
- **RET-01**: Stickiness Factor — % клієнтів, що обирають категорії 3 місяці поспіль
- **RET-02**: Redemption Rate — співвідношення виведеного кешбеку до нарахованого
- **UX-04**: Drill-down з категорії до конкретного мерчанта
- **EXP-01**: PDF export для борду, CSV для аналітиків
- **ALERT-01**: Авто-флаг при ROI < порогу або різкому падінні конверсії

## Out of Scope

- Real-time / T+1 оновлення — немає бекенду в цьому проєкті
- Автентифікація та RBAC на рівні сторінки — не потрібна для демо
